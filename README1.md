# 任务背景

---

本题聚焦于一种“隐性”的信息误导形式：**遗漏式欺骗**。与直接捏造事实不同，遗漏式欺骗通常通过省略关键背景、条件限制、比较基线、事件后果、责任主体或利益相关方观点，使读者在“部分真实信息”的基础上形成偏差理解。伴随着社交媒体和网络新闻的快速传播，互联网上此类隐蔽的误导性表达不断增多。相较于传统的真假新闻检测任务，遗漏式欺骗识别不仅要求模型理解文本中“说了什么”，还需要进一步判断文本“没有说什么”以及这些缺失信息是否足以改变读者对事件的认识，甚至还需要对省略意图进行推理。准确识别传播文本中的遗漏式欺骗及其具体类型，有助于提升对网络误导信息的分析能力，对于信息治理、舆情研判和公共传播安全具有重要意义。本任务将围绕遗漏式欺骗检测与遗漏类型识别展开讨论。

# 任务形式

---

给定一条传播文本 `claim_text` 和一段更完整的背景材料 `context_text`，请判断该传播文本是否构成遗漏式欺骗；如果构成，再识别其主要遗漏类型。

输入格式：

```json
{
  "id": "test_0001",
  "claim_text": "……",
  "context_text": "……"
}
```

输出格式：

```json
{
  "label": "遗漏式欺骗",
  "omission_type": "比较遗漏"
}
```

字段说明：

- `label`：`正常` 或 `遗漏式欺骗`
- `omission_type`：
  - 若 `label=遗漏式欺骗`，则取以下六类之一：
    - `背景遗漏`
    - `复杂遗漏`
    - `比较遗漏`
    - `影响遗漏`
    - `责任遗漏`
    - `利益相关者遗漏`
  - 若 `label=正常`，则取值为 `无`

# 遗漏类型定义

---

1. `背景遗漏`
  - 省略事件前因、时间线、历史背景或政策语境，使读者无法正确理解事件为何发生。
2. `复杂遗漏`
  - 省略条件限制、适用范围、方法细节、不确定性或关键机制，把复杂问题表述得过于简单确定。
3. `比较遗漏`
  - 文本已经给出数据、程度或趋势描述，但缺少同比、环比、历史基线、对照组或横向比较信息，导致意义被夸大或淡化。
4. `影响遗漏`
  - 省略事件后果、潜在风险、波及范围或现实影响，使读者无法判断事件的重要性或严重性。
5. `责任遗漏`
  - 省略事件中应承担责任的主体、决策者、执行者或问责对象，使读者无法判断“谁该负责”。
6. `利益相关者遗漏`
  - 省略受影响群体、相关方立场、利益冲突或多方观点，只呈现单一视角。

---


- `train.json`

`test.json` 中每一行格式如下：

# 任务要求

---

1. 使用给定训练数据完成该任务，方法不限，可以使用大语言模型，基于提示词、SFT 微调、LoRA 等方式。
2. 你可以自行从 `train.json` 中划分训练集和验证集。
3. 对 `test.json` 中的每条样本输出结构化预测结果，至少包含 `label` 与 `omission_type` 两个字段。
4. 建议在报告中说明你的数据处理、指令构造、模型选择、训练设置和误差分析。
5. 完成时间为一周。

# 评价指标

---

本题采用分层评估方式，分别考察一级任务和二级任务的表现：

1. 一级任务：`label` 的 Accuracy / Precision / Recall / F1
2. 二级任务：在 `label=遗漏式欺骗` 的样本上，评估 `omission_type` 的 Accuracy 和 Macro-F1

如果你愿意，也可以设计一个综合指标，但请在报告中说明其计算方式。

# 说明

---

1. 本题重点不在于“真假新闻”本身，而在于识别由**关键事实省略**带来的误导。
2. 同一条样本只标注一个主遗漏类型；若同时存在多类遗漏，以“最主要造成误导的那一类”为准。
3. 判断时请优先关注：传播文本究竟省略了什么信息，以及这种省略主要改变了读者对哪一方面的理解。
